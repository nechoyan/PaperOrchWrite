## Problem Statement

AI coding agents (Claude Code, Cursor, Windsurf, GitHub Copilot) have become the dominant way software is written in 2025-2026. These systems are granted broad filesystem access, execute arbitrary shell commands, read credential stores, and interact with external services through the Model Context Protocol (MCP). Traditional security controls — kernel-level eBPF hooks, cloud DLP, static secret scanners — fail in this environment because they lack semantic context about why the agent issued an action. By the time a kernel hook sees `curl` execute, the agent has already decided to exfiltrate credentials. The gap between agent capability and agent security tooling represents an acute, documented threat with measurable real-world damage: 28.65 million hardcoded secrets leaked on public GitHub in 2025 alone, with Claude Code-assisted commits leaking at 3.2%, more than double the baseline rate.

## Core Hypothesis

The only effective security model for AI coding agents is one that intercepts at the agent's hook layer — the exact point where tool-call decisions are made, before they execute at the OS level. A defense-in-depth architecture composed of three complementary layers addresses the full threat lifecycle: (1) pre-emptive secrets substitution before the agent ever processes real credentials, (2) semantic rule-based runtime enforcement that blocks dangerous tool calls before OS execution, and (3) post-hoc sweep-and-remediate for secrets already exposed in agent caches, transcripts, and logs.

## Proposed Methodology (Detailed Technical Approach)

**Layer 1 — Cloak (Pre-emptive Secrets Prevention)**

Cloak prevents real secret values from entering model context, API transcripts, or session logs. Developers register secrets once into a local store at `~/.prismor/secrets/` (mode 0700/0600). The model references them as `@@SECRET:<name>@@` placeholders in all tool calls. A `PreToolUse` hook substitutes the real value immediately before OS execution; a `PostToolUse` hook uses `sed` to scrub the real value from captured stdout before Claude sees the output. A `UserPromptSubmit` hook detects accidentally pasted credentials (Stripe `sk_live_`, GitHub `ghp_`, AWS `AKIA` patterns) and auto-registers them under a hashed placeholder name, blocking the prompt submission and asking the user to resubmit without the raw secret.

**Layer 2 — Warden (Hook-Layer Runtime Enforcement)**

Warden intercepts at Claude Code's `PreToolUse` and `PostToolUse` hooks — upstream of OS execution — and evaluates every tool call against a YAML policy engine with 25+ default detection rules across 10 categories: destructive commands, secret exfiltration, remote code execution, prompt injection, privilege escalation, denial-of-service, persistence mechanisms, DNS exfiltration, MCP/skill scanner, and network isolation. In `observe` mode, all events are logged to a JSONL audit trail and indexed into a SQLite database. In `enforce` mode, actions matching critical rules are blocked before execution. Rules are expressed as event type + regex pattern + severity + action tuples; project-level overrides merge against the default policy, with non-overridable critical rules that cannot be disabled by any project configuration.

**Layer 3 — Sweep (Post-Hoc Remediation)**

Sweep uses Gitleaks (170+ credential patterns) to scan Claude transcript JSONL files, Cursor/Windsurf/Codex cache directories, and AI tool config directories. In `redact` mode, secrets are replaced in-place with AES-256-CBC encrypted vault backups preserving recovery provenance. In `delete` mode, files containing secrets are removed entirely. In `dry-run` mode, a full report is generated with file path, line number, column, and match type. Configuration files like `.env` are allowlisted since these are developer-intentional credentials managed by Cloak, not accidental leakage.

**Threat Intelligence Feed**

A signed advisory feed (`immunity-feed.json`, 217 advisories as of March 2026) with Ed25519 integrity verification provides runtime correlation. The feed is generated by an automated NVD-fetch pipeline that polls for AI-ecosystem CVEs (LangChain, LlamaIndex, OpenAI, Anthropic, CrewAI) and maps them to Prismor threat categories (prompt_injection, data_exfiltration, unsafe_tool_execution, jailbreak, policy_bypass, dependency_vulnerability, model_denial_of_service).

3. Empirical evidence that pre-emptive secrets cloaking reduces secret leakage risk to near-zero for secrets registered in the store, compared to the 3.2% baseline leak rate for Claude Code-assisted commits
